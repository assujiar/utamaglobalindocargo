# Repo Inventory

## High‑Level Structure

The `utamaglobalindocargo-main` repository is a Next.js 16 application with a typed
TypeScript/Tailwind setup and Supabase back‑end integration.  At the root there are
project documents, configuration files, a `public/` directory with brand assets, a
`src/` folder containing the application code, and a `supabase/` folder
containing SQL migrations for the leads table.  The most relevant top‑level
directories are:

| Path | Purpose |
| --- | --- |
| `src/app/` | Next.js App Router entry points and page routes. Contains the layout component, global CSS, and all route files. |
| `src/components/` | Reusable UI components grouped by domain (layout, sections, services, contact, case studies). |
| `src/lib/` | Helper modules: Supabase server client, analytics helpers, case study data. |
| `src/hooks/` | Custom hooks (UTM capture). |
| `public/` | Favicon and logo SVG. The design currently lacks real photos or OG images. |
| `supabase/` | SQL migration files for the `leads_prospect` table with row‑level security policies. |
| `docs/` | Research reports, previous audits and reviews. This folder also contains markdown deliverables from earlier iterations. |

## Major Page Routes

The application uses the Next.js App Router.  Each folder under `src/app/` becomes a route.

- `/` – Homepage with hero, company introduction, horizontal scrolling services strip, a
  statistics section and a final call‑to‑action.
- `/about` – Company overview page describing who UGC is, who they serve, a summary of
  services, the company’s approach and a final CTA.
- `/contact` – Contact page containing a three‑step multi‑field form with Zod validation
  and a direct contact fallback section.  Form data is submitted to `/api/leads`.
- `/faq` – Frequently asked questions (eight questions and answers) with an accordion UI.
- `/case‑studies` – A grid of anonymized case studies with modal view.  Uses Framer
  Motion for animations.  Content is illustrative and flagged for stakeholder review.
- `/services/domestic‑distribution` – Service page describing Full Truck Load, Less Than
  Truck Load, Full Container Load Domestik (FCL), Less Than Container Load Domestik
  (LCL) and Airfreight Domestik.
- `/services/international‑freight` – Service page describing FCL Export, LCL Export,
  Airfreight Export, FCL Import, LCL Import and Airfreight Import.
- `/services/import‑dtd` – Service page covering FCL Import Door‑to‑Door, LCL Import
  Door‑to‑Door, Airfreight Import Door‑to‑Door and Customs Brokerage.
- `/services/warehousing` – Service page covering General Warehousing, Bonded
  Warehouse, Pick & Pack Fulfillment and Cross‑Docking.
- `/services/project‑cargo` – Service page covering Heavy Lift & Oversize Cargo,
  Dangerous Goods Handling, Temperature Controlled Cargo and Breakbulk &
  Out‑of‑Gauge.
- `/services/blocspace` – Service page covering Blocspace Allocation, Airfreight
  Charter and Sea Freight Charter.  Blocspace was separated from the project cargo
  page in the current build.

Dynamic routes:

- `/api/leads` – API endpoint for submitting contact form data.  Performs server‑side
  Zod validation and inserts leads into the `leads_prospect` table via Supabase.
- `/robots.txt` and `/sitemap.xml` – Generated by files in `src/app/robots.ts` and
  `src/app/sitemap.ts`.

## Major Components

- **Layout components** (`src/components/layout/`):
  - `Header` – Responsive navigation with About and FAQ links and service dropdowns.
  - `Footer` – Four‑column footer grouping service links, company links and contact.
  - `HeroSection` – Homepage hero section containing the headline, sub‑headline,
    description and CTA buttons.  This component currently uses an oversized dark
    background and minimal animation.
  - `SmoothScroller` – Wraps page content and integrates Lenis/GSAP for smooth scroll.
  - `JsonLd` – Adds structured data (Organization schema) to the `<head>`.
  - `ConsoleEasterEgg` – Logs an ASCII cargo ship to the browser console.

- **Homepage sections** (`src/components/sections/`):
  - `ServicesHorizontal` – Horizontal scrolling panel that lists six services with
    labels, short descriptions and links to the detail pages.  Uses GSAP to pin
    panels during scroll on desktop; reverts to vertical layout on mobile.
  - `StatsCounter` – Block intended to display operational metrics such as number
    of countries served or containers moved.  Contains hard‑coded example numbers.

- **Service page template** (`src/components/services/ServicePageLayout`): A generic
  template used by all service pages.  Takes a label, two‑part title, introduction
  and an array of sub‑services.  Each sub‑service includes a name, short name,
  description, key points and recommended use cases.  Alternating dark and light
  backgrounds differentiate sections.

- **Contact form** (`src/components/contact/ContactForm`): A three‑step form
  implemented with `react‑hook‑form` and Zod.  Step 1 captures service interest,
  step 2 captures estimated shipping volume and cargo context, and step 3 captures
  contact details and privacy consent.  Includes a honeypot field for spam
  mitigation, localStorage UTM capture and a direct contact fallback section.

- **Case studies** (`src/components/case‑studies/`): `CaseStudyCard`,
  `CaseStudyGrid` and `CaseStudyModal` compose the case studies page.  Data is
  imported from `src/lib/caseStudiesData.ts`.

## Backend & Data Touchpoints

- **Supabase**: The repository uses Supabase as a backend.  A server‑side
  client (`src/lib/supabaseServer.ts`) is instantiated using the `SUPABASE_SERVICE_ROLE_KEY`.
  All inserts are performed via the API route; there is no client‑side Supabase
  integration.
- **Database**: SQL migrations define a `leads_prospect` table with row‑level
  security and insert policies (`supabase/migrations/00001_create_leads_prospect.sql` and
  `00002_harden_leads_prospect.sql`).  The table stores contact person, company name,
  email, WhatsApp, service interest, operational volume, cargo description,
  origin/destination, timeline, privacy consent and UTM attribution.  Policies
  restrict inserts to authenticated anon users and service role clients.
- **UTM capture**: A custom hook (`src/hooks/useUTMCapture.ts`) persists UTM
  parameters from the query string to localStorage, ensuring attribution data
  accompanies form submissions.
- **Analytics**: `src/lib/analytics.ts` stubs a `trackEvent` function.  No real
  analytics platform is integrated yet.
- **Case studies data**: `src/lib/caseStudiesData.ts` contains six illustrative
  case studies with metrics and results.  A notice warns that these are
  examples and require business review.

## Dependencies & Tooling

The repository uses a modern React/Next stack.  Notable dependencies include:

| Package | Purpose |
| --- | --- |
| `next` (v16) | Framework for React and server‑side rendering |
| `react` / `react‑dom` (v19) | Core UI library |
| `tailwindcss` (v4) | Utility‑first CSS framework; config defined in `postcss.config.mjs` |
| `gsap` / `@gsap/react` | Animation library used for horizontal scrolling panels |
| `framer‑motion` | Animations for the case studies modal |
| `lenis` | Smooth scroll implementation |
| `react‑hook‑form`, `zod`, `@hookform/resolvers` | Form handling and validation |
| `@supabase/supabase‑js` | Supabase client used on the server and in the API route |

Dev dependencies include ESLint (with Next.js preset), TypeScript, and Tailwind/PostCSS.  The repository is configured for strict type checking and includes a basic GitHub Actions CI workflow (`.github/workflows/ci.yml`) that runs linting, type checking and the production build.
